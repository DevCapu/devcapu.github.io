9:T590,<h2>Core Concepts</h2>
<p>TypeScript adds a static type system on top of JavaScript. Types are erased at compile time — the output is plain JS.</p>
<h2>Utility Types</h2>
<h3><code>Partial&#x3C;T></code></h3>
<p>Makes all properties optional:</p>
<pre><code class="language-ts">interface User { id: number; name: string }
type PartialUser = Partial&#x3C;User> // { id?: number; name?: string }
</code></pre>
<h3><code>Pick&#x3C;T, K></code> and <code>Omit&#x3C;T, K></code></h3>
<pre><code class="language-ts">type UserPreview = Pick&#x3C;User, 'id' | 'name'>
type UserWithoutId = Omit&#x3C;User, 'id'>
</code></pre>
<h3><code>Record&#x3C;K, V></code></h3>
<pre><code class="language-ts">const map: Record&#x3C;string, number> = { a: 1, b: 2 }
</code></pre>
<h2>Generics</h2>
<pre><code class="language-ts">function identity&#x3C;T>(value: T): T {
  return value
}
</code></pre>
<h2>Type Narrowing</h2>
<pre><code class="language-ts">function process(input: string | number) {
  if (typeof input === 'string') {
    // input is narrowed to string here
    return input.toUpperCase()
  }
  return input * 2
}
</code></pre>
<h2><code>satisfies</code> operator</h2>
<p>Validates a value against a type without widening it:</p>
<pre><code class="language-ts">const config = {
  port: 3000,
  host: 'localhost',
} satisfies Record&#x3C;string, string | number>
// config.port is still number, not string | number
</code></pre>
